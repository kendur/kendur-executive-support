# Roadmap

## Phase 0 — Foundation *(current)*

- [x] Create public repository and documentation foundation
- [x] Define Notion-centered, AI-optional architecture
- [x] Define Personal and Work profiles
- [x] Define open community and future service model
- [x] Define capture-source-agnostic ingestion direction
- [x] Record initial project history and ADRs
- [ ] Add behavioral, UX, platform, and competitor research
- [ ] Audit the current private Notion system
- [ ] Select the permanent product name and licenses

## Phase 1 — Manual Notion prototype

**Goal:** Prove the core workflow without AI or external automation.

- [ ] Define the minimum data model
- [ ] Build Capture, Clarify, Now, Today, Project, Waiting/Blocked, and Recovery views
- [ ] Keep due dates separate from scheduled work
- [ ] Add source provenance and restart cues
- [ ] Support separate Personal and Work profiles with a unified view
- [ ] Test spontaneous-conversation, meeting-follow-up, and manual-task workflows
- [ ] Run a no-documentation usability test
- [ ] Use the prototype daily and record friction

## Phase 2 — Community release

- [ ] Publish the complete manual Notion template
- [ ] Publish the AI Guide Pack
- [ ] Add CC BY 4.0 content licensing and attribution guidance
- [ ] Publish update and migration instructions
- [ ] Establish community feedback and release testing

## Phase 3 — Replaceable automation adapters

- [ ] Finalize the canonical capture contract
- [ ] Add universal manual, email, folder, and file ingest
- [ ] Add Tasker/mobile capture

- [ ] Document deterministic workflow behavior independently from the platform
- [ ] Add Apache 2.0 software licensing

## Phase 4 — Optional intelligence

- [ ] Define provider-agnostic capability interfaces
- [ ] Add action and decision extraction

- [ ] Add progressive, skill-aware onboarding
- [ ] Add local and hosted provider examples
- [ ] Validate graduated autonomy and provenance handling
