# Contributing to the Executive Support System

Thank you for contributing feedback, research, documentation, design proposals, templates, tests, or code.

## Ground rules

- Be respectful, constructive, and direct.
- This project has a single primary maintainer; response times may vary.
- Human maintainers remain accountable for accepted work.
- AI-assisted contributions are permitted and must be disclosed in the pull request.
- Significant direction changes should begin with an issue and, when accepted, an ADR.

## Current contribution areas

- documentation corrections;
- behavioral, UX, platform, and market research;
- workflow and data-model proposals;
- usability test cases;
- ADR proposals;
- issue reports for ambiguity, friction, or inconsistency.

Later phases will add Notion template changes, automation adapters, AI integrations, and implementation tests.

## Submitting changes

1. Open an issue for a significant change.
2. Fork the repository and create a descriptive branch.
3. Make a focused change.
4. Open a pull request using the template.
5. Explain the problem, change, evidence, and trade-offs.

## Documentation standards

- Use plain language and sentence-case headings.
- Prefer short paragraphs and explicit wording.
- Use **Profile** for top-level separation and **Personal** and **Work** as the public defaults.
- Do not include employer names, confidential information, real recordings, credentials, or personal data.
- Do not make diagnostic, treatment, or therapeutic claims.
- Evidence-based behavioral and interaction-design research is welcome when accurately cited.
- Do not assume a specific LLM, automation platform, recorder, calendar, or storage provider.
- Preserve project history; supersede accepted ADRs rather than rewriting them.

## AI-assisted contributions

Disclose material AI assistance and describe its role. Contributors remain responsible for understanding, testing, sourcing, and reviewing their changes.

## Conduct

Be constructive. Focus criticism on ideas, evidence, mechanics, and outcomes.

## Licensing status

No license files have been assigned yet. Until licensing is published, all rights remain with the repository owner. Licensing and contribution terms will be clarified before external code or template contributions are accepted at scale.
